10.1

ウェブの基本構造

10.1.1 URL とドメイン名の構造

ウェブ上の資源にはそれぞれ URL（Uniform Resource Locator）が割り当てられており、いわば「インターネット上の住所」の役割を果たしています。典型的な URL は https://www.example.com/info/page1 のような形式で、プロトコル（https）、ドメイン名（www.example.com）、パス（/info/page1）から構成されます。ドメイン名は DNS（ドメインネームシステム）によって IP アドレスと結び付けられています。

10.1.2 HTML ページの構造

ウェブページは通常 HTML（HyperText Markup Language）というマークアップ言語で記述されています。HTML ではタグ（要素）を用いて文書の構造をマークアップし、見出しや段落、リンク、画像などの要素を階層的に配置します。HTML 文書は基本的に <html> 要素で囲まれ、その内部は大きくヘッド部とボディ部に分かれます。

ヘッド部にはページのタイトル（<title>タグ）や文字エンコーディングの指定、メタデータ、スタイルシート（CSS）やスクリプト（JavaScript）へのリンクが置かれます。ボディ部（<body>）が実際にユーザーに表示されるコンテンツです。

HTML5 以降では、ページの論理的な領域を示すセマンティック要素も導入されました。例えばヘッダーに <header>、ナビゲーションメニューに <nav>、主要コンテンツ部分に <main>、各記事に <article>、フッターに <footer> といった具合に、意味を持ったブロック要素を使ってマークアップできます。

10.1.3 リンク構造とネットワーク性

HTML でもっとも特徴的な要素の一つがハイパーリンク（<a href="...">タグ）です。リンクをクリックすると別のページに移動でき、ウェブの文書同士を結びつける役割を果たします。リンクには大きく分けて内部リンク（同じサイト内）と外部リンク（別のサイトへ）の2種類があります。

このようにリンクによってウェブ上のページは相互に繋がっており、その集合体は巨大なネットワーク（グラフ）を形成しています。各ウェブページがノード（頂点）に相当し、リンクがノード同士を結ぶエッジ（辺）として機能する構造で、これを特にウェブグラフと呼びます。

リンク構造はウェブの「つながり」を定義する重要な要素であり、その解析から多くの知見が得られています。有名な PageRank に代表されるアルゴリズムでは「多くの他ページからリンクされているページほど重要である」という考え方に基づきページの重要度を推定しています。ウェブにおけるリンク構造は単なるナビゲーション手段に留まらず、ウェブ全体を一つのネットワークとして機能させる根幹であり、その分析には社会ネットワーク論的なアプローチも有効です。

10.2

Hyphe の紹介

ウェブ上の情報を収集・解析する専門的なツールの一つに Hyphe（イフ）があります。Hyphe はフランスの Sciences Po（シアンスポ）大学メディアラボによって開発されたオープンソースのウェブクローラで、社会科学や人文学の研究者がウェブ上のコーパスを構築・分析するために設計されています。

一般的なクローラが自動で大量のページを巡回する「ブラックボックス」的な動作をするのに対し、Hyphe は研究者が主導してキュレーション指向のアプローチを採用している点が理論的特徴です。具体的には、ユーザーが関心のあるウェブページやサイトを少しずつ指定し、それらを起点に関連ページをクロールし、得られたリンク先をまた取捨選択し……というように、ステップバイステップで質的調査の試行錯誤を組み込んだウェブ収集ができるのです。

ウェブエンティティ（web entity）

Hyphe でもう一つ重要な概念がウェブエンティティです。通常のクローラでは各 URL（ページ）ごとにデータを扱いますが、Hyphe では研究の都合に合わせて複数のページを一つの「エンティティ」として束ねて扱うことができます。例えば「同じドメイン内のページはまとめて1エンティティ（＝サイト単位）」としたり、あるいはサブドメインごとに分けたり、特定の複数サイトをまとめて一つのエンティティとみなす、といった柔軟な単位設定が可能です。収集が完了すると、ウェブエンティティ同士のハイパーリンクネットワークが構築されます。

活用の可能性

Hyphe は人文学・社会科学の研究において様々な活用可能性を持ちます。例えばある社会問題についてウェブ上でどのような議論が交わされ、どの団体・個人・メディアが関与しているかを把握するには、関連するウェブサイト群を網羅的に集めて相互のリンク関係を見るのが有効です。Hyphe で得たネットワーク図からは、議論に参加するコミュニティの構造（クラスターの分布や相互の分断・結び付き）が視覚的に読み取れます。さらに Hyphe は近年ウェブアーカイブ（Internet Archive の Wayback Machine やフランス国立図書館のウェブ保存）にも対応しつつあり、過去のある時点でのウェブの状態を再現し、その当時の議論やネットワーク構造を分析する、といった時系列比較にも道を開いています。

10.3

WordPress によるウェブページ制作

ウェブページを実際に構築・公開する方法はいくつかありますが、本節では現在広く普及している WordPress を用いたサイト制作について説明します。WordPress はブログやサイトを構築・運用できるオープンソースの CMS（コンテンツ管理システム）で、全世界のウェブサイトのかなりの割合で採用されています。

WordPress.com（クラウド型）

サーバーやソフトウェアを自分で用意しなくても、ウェブ上のアカウント登録だけで WordPress サイトを開設できるクラウドサービスです。初心者にとって手軽に始められる利点がありますが、無料プランではテーマやプラグインに制限があり、カスタマイズの自由度は高くありません。「手軽さ」と「カスタマイズ自由度」のトレードオフがあります。

WordPress.org（自前設置型）

自分で用意したサーバー環境に WordPress のソフトウェア一式をインストールしてサイトを構築します。初期設定の手間はかかりますが、サイトの構成や機能を自由にカスタマイズできるのが大きな魅力です。研究プロジェクトの特設サイトなどで凝ったデザインや特殊な機能を実装したい場合に望ましいでしょう。

WordPress.org での構築手順

1.
サーバー契約：レンタルサーバー会社と契約。日本国内ではロリポップ!やさくらインターネット、エックスサーバー、ConoHa WING 等が WordPress 利用に適しています。PHP とデータベース（MySQL 等）の利用可能を確認します。
2.
ドメイン取得と SSL 設定：サイトの URL となる独自ドメインを取得します。SSL（HTTPS）の設定も行い、通信を暗号化してセキュリティを確保します。
3.
WordPress のインストールと初期設定：管理画面で「WordPress インストール」を選び、サイトタイトルや管理者ユーザー名・パスワード、使用する独自ドメイン等の必要事項を入力するだけで自動でセットアップが完了します。

10.3.3 コンテンツ設計

良いウェブサイトを作るには、闇雲にページを作り始める前にサイト全体の構成（情報設計）をしっかり考えることが大切です。人文学系の研究者が自身のサイトを作る際によくある目的として、(a) 自身の研究や業績を紹介する個人サイト、(b) 特定の研究プロジェクトの公式サイト（ポータル）、(c) 論文や資料のアーカイブサイト、などが挙げられます。

研究紹介サイト（個人サイト）

研究テーマ、業績一覧（論文PDF・DOI付き）、プロフィール（写真・略歴・所属学会）、連絡先。自分の専門分野や関心を外部の人に伝える名刺代わりとなるサイト。

プロジェクトポータルサイト

プロジェクトの背景・目的、メンバー一覧、活動報告（ブログ形式）、成果物公開ページ、イベント告知ページ。SNSアカウントへのリンクも設置。

論文アーカイブサイト

論文や記事の一覧（発表年度別・テーマ別・著者別など複数の切り口で誘導）。各論文の個別ページにタイトル、著者、掲載誌、要旨、参考文献リスト、PDF フルテキストへのリンクを掲載。

重要なのは、ユーザー（訪問者）が求める情報にできるだけ少ないクリックで到達できるようにすることです。一般に「3クリック以内で目的の内容にアクセスできる」ことが望ましいとされます。ナビゲーション設計や内部リンクの配置、モバイル対応（レスポンシブデザイン）なども重要なポイントです。

10.3.4 最低限のデザイン原則

  • ●
    配色（カラーリング）：背景色と文字色のコントラストに注意。研究内容を伝えるサイトでは落ち着いたトーンで信頼感を与える配色が適しています。アクセントカラーは1色程度に絞ると統一感が出ます。色覚多様性への配慮も念頭に置くとベターです。
  • ●
    タイポグラフィ（字体・文字組み）：日本語であればゴシック体（サンセリフ系）か明朝体（セリフ系）の Web フォントを利用。本文サイズはPC表示なら16px程度を下回らないようにし、行間（ラインハイト）も十分取り、詰まり過ぎない段落のバランスも大事です。
  • ●
    モバイル対応：レスポンシブデザインは現代のウェブ制作で不可欠です。スマートフォンでの見え方・操作感をテストしましょう。Google の検索順位決定もモバイル版サイトの出来を重視するモバイルファーストインデックスに移行しています。
  • ●
    テーマの選定：WordPress の強みの一つがテーマを適用できることです。サイトの目的と求めるデザイン・機能に合ったテーマを選ぶことが重要です。モバイル対応や表示速度もテーマ選びの重要な観点です。
10.4

Notion でポートフォリオを公開する

10.4.1 Notion でポートフォリオを作成する意義

Notion はドキュメント作成やタスク管理、データベース管理など多機能な「オールインワン」ワークスペースですが、そのページはワンクリックで魅力的な Web サイトに変換できる点が大きな特徴です。ポートフォリオとは、自分のこれまでの研究・学習・活動やスキルなどをまとめた自己紹介・業績集のことです。大学院生にとってのポートフォリオは、自身の歩んできた道のりや将来のビジョンを整理し、第三者に伝える自己PRツールであり、同時に自己省察の機会にもなります。

Notion を使うメリットは、手軽さと自由度の高さ（テンプレートを活用したスピーディーなページ制作）、更新・編集のしやすさ（Web 上でそのまま編集可能）、コンテンツの一元化（散らばっている成果物や情報を1つのページにまとめて表示・管理できる）、公開・共有の容易さ（ワンクリックで Web 公開可能）、そして無料で使えることです。

10.4.2 コンテンツ構造化の実践方法

Notion ではコンテンツをブロックと呼ばれる単位で管理します。ポートフォリオに載せる内容は大きく5つに分けて考えると整理しやすいです。

1. プロフィール

氏名・所属、自己紹介文

2. キャリア

過去〜現在〜未来の歩み

3. 研究

テーマ・概要・プロセス

4. スキル・実績

学歴・スキル・業績

5. 私生活・趣味

パーソナルな側面

ページとサブページによる構成、テキストの装飾とレイアウト（見出し・箇条書き・番号付きリスト）、視覚要素の活用（画像や図表、引用ブロック、コールアウトブロック）、リンクや埋め込み（YouTube 動画、Google マップ等のプレビュー埋め込み）、テンプレートの活用などが実践のポイントです。

10.4.3 Notion データベース機能の紹介と応用

Notion の強力な機能の一つにデータベースがあります。用途に応じて以下のような表示形式があります。

テーブルビュー：表形式 リストビュー：縦長リスト ボードビュー：カンバン方式 カレンダービュー：日付基準 ギャラリービュー：カード形式（サムネイル画像付き） タイムラインビュー：ガントチャート風

例えば論文一覧データベースなら、「タイトル」「掲載年（日付）」「学会名/誌名（テキスト）」「分類（選択肢あるいはタグ）」などのプロパティを設定しておけば、後から発表年順やカテゴリ別にソート・フィルタして表示するといった操作も簡単です。ギャラリービューは特にポートフォリオで視覚的に訴えたい場合におすすめです。

10.4.4 Notion ページを Web 上に公開する手順

完成した Notion ページは、設定を変更することでウェブ上に公開できます。手順は非常に簡単です。

1.公開したい Notion ページを開き、画面右上にある「共有」ボタンをクリックします。
2.共有メニュー内で「Web 公開」タブに切り替え、「Web で公開」スイッチをオンにします。公開用の URL が自動生成されます。
3.表示された公開 URL をコピーし、閲覧を許可したい相手にそのリンクを共有します。リンクを知っている人であれば、Notion アカウントを持っていなくてもブラウザ経由でページを閲覧可能です。

各種公開設定として、検索エンジンでの表示可否（「ネット検索を許可」オプション）、編集・コメント権限、複製の許可があります。限定公開にしたい場合は「ネット検索を許可」をオフにしておくのが基本です。

10.5

Notion ポートフォリオ：実際の設計と制作

ポートフォリオの5つのセクション（プロフィール、キャリアとビジョン、研究、スキル・実績、私生活・趣味）それぞれについて、何を書くべきかを具体的にイメージできるよう質問形式のガイドと例を交えながら解説しています。

プロフィール

氏名・所属・専門分野を簡潔に。1〜3行程度の自己紹介文。可能であれば顔写真やイメージ画像も配置。ビジュアルとテキストを組み合わせることで第一印象を強め、印象付ける効果があります。

キャリアとビジョン

【過去】【現在】【未来】の三つの観点から整理。「なぜ大学院に進学したのか？」「現在取り組んでいることや直近の目標」「5年後、10年後のビジョン」。自分の考えや感情を織り交ぜることが大切。

研究

研究テーマ・概要（専門用語を避け平易な言葉で説明）、研究の意義・面白さ（自分なりの視点で語る）、これまでの成果、研究プロセス・スタイル（カバンの中身、お気に入りの研究道具、デジタルツール活用、作業場所・環境など）。

スキル・実績

学歴・経歴、専門スキル（言語・データ分析・資料調査など）、資格・免許、研究業績、受賞歴・表彰、その他の活動実績。カテゴリ分けして整理すると見やすくなります。

私生活・趣味

パーソナルな側面を見せるセクション。食事スタイル、リフレッシュ方法、一日のルーティン、作業効率アップの工夫、おすすめの本・映画・動画、アルバイト経験、お気に入りの場所など。「等身大の自分」を見せましょう。ただしプライバシーには注意。
